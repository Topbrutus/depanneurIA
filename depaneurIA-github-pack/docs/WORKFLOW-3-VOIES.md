# Workflow à trois — toi / Copilot / ChatGPT

## Source unique

Le dépôt GitHub est la source unique de vérité.

## Répartition

### Toi
- tu choisis la direction ;
- tu valides les priorités ;
- tu décides quand fusionner.

### Copilot
- il exécute une tâche précise ;
- il ouvre une PR ;
- il doit respecter le bloc actif.

### ChatGPT
- il découpe le travail ;
- il garde la cohérence ;
- il vérifie où vous êtes rendus.

## Ordre recommandé

1. Lire `docs/STATE.md`
2. Choisir le bloc actif dans `docs/TASK-BLOCKS.md`
3. Ouvrir l’issue correspondante
4. Faire le travail
5. Ouvrir une PR
6. Mettre à jour `STATE.md`
7. Cocher les lignes terminées

## Pourquoi ce système marche

Parce que l’état n’est pas dans la tête d’un outil.
Il est écrit noir sur blanc dans le dépôt.
