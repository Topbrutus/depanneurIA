# DEP-0121 à DEP-0127 — Décisions techniques initiales

## Périmètre

Acter les choix socle du stack pour la V1 : langage principal, frameworks front/back, modèle de dépôt, gestionnaire de paquets et versions minimales de Node.js et TypeScript.

## Décisions

- **DEP-0121 — Langage principal : TypeScript**
  - TypeScript devient le langage par défaut pour le code applicatif et les outils internes, afin de garantir le typage, l’autocomplétion et la cohérence sur l’ensemble du projet.

- **DEP-0122 — Interface web : React**
  - React est retenu pour le frontend web, avec usage systématique de TypeScript. Ce choix aligne l’interface avec l’écosystème JS/TS, facilite le partage de composants et bénéficie d’un écosystème mature (hooks, routeurs, tests).

- **DEP-0123 — Backend : Node.js**
  - Le backend s’appuie sur Node.js (TypeScript), pour rester dans une stack homogène JS/TS, profiter de l’écosystème npm (http, web, outillage) et simplifier le partage de modèles/types entre front et back.

- **DEP-0124 — Modèle de dépôt : monorepo**
  - Un monorepo unique est adopté pour héberger toutes les apps (`apps/`) et librairies partagées (`packages/`). Objectifs : partage de types, versionnement aligné, revue unique et outillage commun (lint, format, tests).

- **DEP-0125 — Gestionnaire de paquets : npm (workspaces)**
  - Un seul gestionnaire : `npm`, en mode workspaces pour le monorepo. Motifs : outil natif avec Node LTS, évite l’ajout d’une dépendance externe, compatibilité CI simple et onboarding sans installation supplémentaire.

- **DEP-0126 — Version minimale Node.js**
  - Version minimale supportée : **Node.js 20 (LTS)**. Raison : support long terme, perfs et fonctionnalités récentes (fetch natif, permissions expérimentales), npm 10 inclus pour gérer les workspaces.

- **DEP-0127 — Version minimale TypeScript**
  - Version minimale supportée : **TypeScript 5.3**. Raison : prise en charge complète des libs Node 20, améliorations de rétrécissement et des générics, compatibilité avec l’écosystème React récent.

## Impacts et prochaines étapes

- Aucun fichier de configuration n’est modifié ici ; les tsconfig, scripts npm et outillage qualité seront ajoutés dans les blocs suivants (DEP-0128+).
- Les futurs modules devront respecter ces choix (TS/React/Node) et être structurés dans le monorepo `apps/` et `packages/` sous npm workspaces.
